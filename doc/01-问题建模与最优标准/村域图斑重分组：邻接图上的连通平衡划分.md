这个问题的关键，不是“怎么切一个大多边形”，而是“怎么在**不切坏原子单元（图斑）**的前提下做**连通、平衡、无拓扑问题**的分组”。一旦你把模型改对，后面很多约束都会从“事后检查”变成“构造时天然满足”。

我先给结论：
如果你的硬约束真的是“不能把已有图斑一分为二”，那么最自然的建模不是直接对村边界做几何切割，而是把每个图斑当成不可再分的原子单元，先构建“图斑邻接图”，再做“连通平衡划分”，最后把每组图斑做 union 还原成子村边界。这样做时，子村边界之间的分界线天然落在原有图斑边界上，不会把图斑切开，也更容易保证拓扑合法。

先把问题拆开看。

## 1. 从极简模型推到真实场景

### 1.1 极简模型：一维连续分段

先假设不是二维图斑，而是一排排好序的小区间。现在要求把这一维序列切成若干段，每段元素数不超过阈值 `T`。

如果目标只有“每段不超过 `T`”，那最优块数就是 `k = ceil(n / T)`。
如果还要“尽量均匀”，那最自然的目标就是让每段大小尽量接近 `n / k`。在一维有顺序约束时，这类问题通常可以用贪心或 DP 做到很干净，因为“连续”本身就替你保证了连通性。

这一层你应该抓住两个点：

第一，**最优之前必须先定义目标**。
如果只说“每块都小于阈值”，那把每个图斑单独分一块也满足约束，但显然不是你想要的。所以必须再加一个目标，例如：

1. 先让块数最少，即固定 `k = ceil(N / T)`；
2. 在块数固定下，再尽量均匀。

第二，**顺序一旦消失，难度会陡增**。
一维里有天然顺序；二维里没有。

### 1.2 二维网格模型：难点开始出现

把一维换成二维网格后，每个单元和上下左右相邻。现在要把网格分成若干个**连通**子集，并且每个子集大小不超过 `T`，还希望尽量均匀。

这时问题本质已经变成了**连通平衡图划分**。文献里对 connected k-partition、balanced connected partition 的研究很成熟，而且这类问题在一般图上是 NP-hard；Moura 等 2023 的论文也明确指出，带权 connected k-partition 的 min-max / max-min 版本对任意固定 `k ≥ 2` 都是 NP-hard。也就是说，你一旦进入二维连通划分，想要对 10 万级图斑求全局精确最优，基本就不现实了。([数学、统计与计算机科学院][1])

### 1.3 真实场景：几何层 + 图层

你的真实问题其实有两层：

第一层是**组合优化层**：
哪些图斑分到同一组。

第二层是**几何重建层**：
每组图斑如何重新变成一个合法子边界。

这两层必须分开想。
经典的 polygon partition 算法，比如 CGAL 的 `Partition_2`，研究的是把一个**简单多边形**分成 y-monotone 或 convex 子多边形；它的输入假设是边界点序列，而不是“内部有大量不可切分的图斑”。CGAL 文档里也明确写了，这类算法是在简单多边形上做分解，甚至最优凸分解的 Greene 动态规划最坏复杂度就是 `O(n^4)` 时间、`O(n^3)` 空间。它对“不能切内部图斑”这个业务约束并不直接适配。([CGAL][2])

所以，你这个问题最合适的抽象不是“多边形切割”，而是：

**把图斑当作面单元，构建平面覆盖（coverage）的对偶图，然后做连通平衡划分。**

## 2. 正确建模：把图斑变成图

设每个图斑是一个节点 `v`。

如果两个图斑共享一段边界（不是只在一点接触），就在它们之间连一条边 `e`。
边权可以设为它们的**共边长度**，因为切断一条长共边，通常意味着更“伤形状”；切断短边通常更自然。

节点权重设为：

- 图斑节点：`1`
- 如果村边界内存在未被图斑覆盖的残余面，可把这些残余面也作为节点加入，权重设为 `0`

于是问题变成：

把这个平面图划分成 `k` 个连通块，使得：

- 每个块的图斑数 `<= T`
- 所有块覆盖整个村域
- 块之间不重叠
- 尽量均衡
- 尽量少切断强邻接边

注意这里非常关键的一点：

**如果你不先固定 `k` 或不惩罚块数，“最优”是没有定义的。**

因为：

- 只要求 `<= T`，那每个图斑单独成块永远可行；
- 如果又说“尽量均匀”，那大量小块反而更容易“均匀”。

所以，工程上通常先固定：
`k = ceil(N / T)`

然后在这个 `k` 下优化均衡和切割质量。这个思路和图划分工具的默认目标也是一致的：给定 `k`，尽量平衡各块大小，同时最小化 cut。KaHIP 的 README 也明确把问题表述为把图的节点集划分成 `k` 个“equally sized blocks”，同时最小化块间边；它从 v3.25 开始还加入了实验性的 `--connected_blocks`，用于强制每个块都连通。METIS 则是更经典的多层递归二分 / k-way / multi-constraint partitioning 框架，但默认不强调“每块连通”这个额外约束。([GitHub][3])

## 3. 方案对比：哪些能做，哪些不适合

### 3.1 直接几何切线 / kd-tree / quadtree

思路是按空间位置切村边界，比如横切、竖切、递归二分。

优点是简单，通常 `O(n log n)` 就能做。
问题是它天然容易把图斑切开。你可以再做“吸附到边界”的修补，但修补一做，问题又回到了图划分本质。

所以这个方案适合“点数据/矩形网格/允许切对象”，不适合你这种“图斑不可切”的业务。

### 3.2 Hilbert / Z-order / 空间填充曲线排序后分桶

先按空间填充曲线给图斑排序，再按顺序分组。

优点是实现非常快，空间局部性通常比普通排序好。
问题是：

- 只能近似连通，不保证连通；
- 容易出现蛇形分块；
- 最后 union 可能得到 MultiPolygon 或细长怪形。

可用于粗分或预聚类，不适合作为最终答案。

### 3.3 贪心区域生长（region growing）

从若干种子图斑出发，同时向外长，直到每组达到目标大小。

优点是天然不会切图斑，也容易保持连通。
问题是强依赖种子选取和生长顺序，质量波动较大，容易形成局部最优。

这是一个不错的工程启发式，但不是我最推荐的第一版。

### 3.4 邻接图上的递归平衡二分

先构建图斑邻接图，再做递归二分，直到每块都不超过阈值。

优点是：

- 约束最贴合业务；
- 不切图斑；
- 连通性可直接在图上保证；
- 最终几何重建简单。

如果再配合高质量图划分器，例如 KaHIP，通常这是最像“工程最优”的方向。([GitHub][3])

### 3.5 ILP / CP-SAT 精确优化

把每个图斑属于哪个块建成 0/1 变量，再加连通性、容量、cut 等约束求解。

优点是目标可控，理论上能出真正最优。
问题是规模一大就很难扛，10k 到 100k 基本不现实，除非你先强力 coarsen 成很小的超点图，再在粗图上精确求解。

所以它更适合作为“小规模基准”或“coarsen 后 refinement”。

## 4. 怎么判断“最优切分方案”

这里必须先把“最优”说清楚。对你的场景，我建议用**词典序目标**：

第一层，硬约束，必须 100% 满足：

1. 所有原图斑都且仅属于一个子集
2. 每个子集内图斑数 `<= T`
3. 每个子集在邻接图上连通
4. 几何上子边界合法，子边界之间 interior-disjoint
5. 所有子边界的并集等于原村边界（若原图斑未完全覆盖村域，则需要把残余面也纳入建模）

第二层，主目标：

6. 块数最少，即 `k = ceil(N / T)`

第三层，次目标：

7. 均衡性最好，比如最小化
   - `max_i count_i`
   - 或 `max_i |count_i - N/k|`
   - 或标准差

第四层，形状与切割质量：

8. 最小化被切断的共边总长度
9. 尽量减小外边界增长，尽量避免细长条
10. 尽量减少洞、飞地、极细连接

如果你只给一个综合分数，我建议：

`score = A * blockCountPenalty + B * imbalancePenalty + C * cutLength + D * shapePenalty`

但工程上更稳的是**词典序**，因为这样不会出现“为了更圆一点，结果多切一块”的反直觉结果。

## 5. 我推荐的方案

如果你要的是“先做出一版可靠、可解释、纯 Java、便于内化”的方案，我建议：

### 方案 A：纯 Java + JTS 的“邻接图 + 最大生成树 + 递归平衡切树”

这不是理论最优，但它有几个很强的优点：

- 完全不切图斑
- 连通性是构造保证的，不是事后碰运气
- 代码可控，不依赖本地 C++ 图划分器
- 复杂度和实现难度都比较适中
- 很适合作为你理解这个问题的第一版基线

核心思想很简单：

1. 图斑构图
2. 以共边长度为边权，构建最大生成树
3. 递归找一条“弱边”切开，使两边图斑数量尽量接近目标，并且未来还能继续分到阈值以下
4. 每次切树边，天然得到两个连通块
5. 递归直到每块 `<= T`
6. 每块内部图斑做 union 还原成子边界

这里最大生成树的直觉是：

- 共享边长越长，两个图斑“绑定”越强
- 最大生成树尽量把强关系留在树里
- 最后切树时优先切短弱边，通常更自然

这是一种很典型的“先把难问题降到树上”的思想。

JTS 在这个方案里主要负责三件事：

- `STRtree`：建空间索引，快速找可能相邻的图斑。JTS 文档说明它是基于 STR 的 query-only R-tree，适合二维空间数据；第一次 query 时完成构建，之后不能再继续 add。([locationtech.github.io][4])
- `CoverageUnion`：把同一组图斑高效 union 成子边界。JTS 文档明确说它针对 valid coverage 更高效，但前提是输入必须 vector-clean 且 interior-disjoint。([locationtech.github.io][5])
- `Geometry.isValid` / `IsValidOp`：做几何有效性校验，JTS 文档说明它按 OGC Simple Features 语义判断几何是否 valid。([locationtech.github.io][6])

下面给你一版单文件、可直接理解和改造的 Java 代码。
这版代码优先保证思路清晰和约束正确，邻接构建采用通用的 STRtree + 共边判定；如果你后续确认“共边坐标完全一致”，生产版我建议把邻接构建替换成“边段哈希法”，那会更快。

```java
import org.locationtech.jts.geom.*;
import org.locationtech.jts.index.strtree.STRtree;
import org.locationtech.jts.operation.overlayng.CoverageUnion;
import org.locationtech.jts.operation.union.UnaryUnionOp;
import org.locationtech.jts.operation.valid.IsValidOp;

import java.util.*;
import java.util.stream.Collectors;
import java.util.stream.IntStream;

/**
 * 村边界切分：
 * 1) 图斑作为不可切原子单元
 * 2) 构建图斑邻接图（共享边界长度 > 0）
 * 3) 基于最大生成树递归切分，保证每块连通
 * 4) 每块图斑数 <= threshold
 * 5) 最终把每块 union 成子边界
 *
 * 依赖：
 *   org.locationtech.jts:jts-core:1.20.x
 */
public final class VillagePartitioner {

    private static final double EPS = 1e-8;

    private VillagePartitioner() {
    }

    public static final class Region {
        public final int regionId;
        public final List<Integer> plotIds; // 原始图斑下标
        public final Geometry geometry;
        public final int plotCount;

        public Region(int regionId, List<Integer> plotIds, Geometry geometry, int plotCount) {
            this.regionId = regionId;
            this.plotIds = plotIds;
            this.geometry = geometry;
            this.plotCount = plotCount;
        }

        @Override
        public String toString() {
            return "Region{id=" + regionId + ", plotCount=" + plotCount + ", plotIds=" + plotIds.size() + "}";
        }
    }

    public static final class ValidationReport {
        public final List<String> errors = new ArrayList<>();

        public boolean ok() {
            return errors.isEmpty();
        }

        @Override
        public String toString() {
            return ok() ? "OK" : String.join("\n", errors);
        }
    }

    private static final class Cell {
        final int id;
        final Geometry geometry;
        final int plotCount;       // 图斑=1，残余面=0
        final int originalPlotId;  // 残余面=-1
        final Coordinate centroid;

        Cell(int id, Geometry geometry, int plotCount, int originalPlotId) {
            this.id = id;
            this.geometry = geometry;
            this.plotCount = plotCount;
            this.originalPlotId = originalPlotId;
            this.centroid = geometry.getCentroid().getCoordinate();
        }
    }

    private static final class Edge {
        final int u;
        final int v;
        final double sharedLength;

        Edge(int u, int v, double sharedLength) {
            this.u = u;
            this.v = v;
            this.sharedLength = sharedLength;
        }
    }

    private static final class TreeEdge {
        final int to;
        final double sharedLength;

        TreeEdge(int to, double sharedLength) {
            this.to = to;
            this.sharedLength = sharedLength;
        }
    }

    private static final class BlockSplit {
        final List<Integer> leftNodes;
        final List<Integer> rightNodes;
        final int leftWeight;
        final int rightWeight;

        BlockSplit(List<Integer> leftNodes, List<Integer> rightNodes, int leftWeight, int rightWeight) {
            this.leftNodes = leftNodes;
            this.rightNodes = rightNodes;
            this.leftWeight = leftWeight;
            this.rightWeight = rightWeight;
        }
    }

    private static final class CandidateCut {
        final boolean feasible;
        final double imbalance;
        final double cutEdgeWeight; // 越小越好
        final int childPos;

        CandidateCut(boolean feasible, double imbalance, double cutEdgeWeight, int childPos) {
            this.feasible = feasible;
            this.imbalance = imbalance;
            this.cutEdgeWeight = cutEdgeWeight;
            this.childPos = childPos;
        }

        boolean betterThan(CandidateCut other) {
            if (other == null) return true;
            if (this.feasible != other.feasible) return this.feasible;
            int c1 = Double.compare(this.imbalance, other.imbalance);
            if (c1 != 0) return c1 < 0;
            int c2 = Double.compare(this.cutEdgeWeight, other.cutEdgeWeight);
            if (c2 != 0) return c2 < 0;
            return this.childPos < other.childPos;
        }
    }

    private static final class DSU {
        final int[] parent;
        final int[] rank;

        DSU(int n) {
            this.parent = new int[n];
            this.rank = new int[n];
            for (int i = 0; i < n; i++) parent[i] = i;
        }

        int find(int x) {
            if (parent[x] != x) parent[x] = find(parent[x]);
            return parent[x];
        }

        boolean union(int a, int b) {
            int ra = find(a);
            int rb = find(b);
            if (ra == rb) return false;
            if (rank[ra] < rank[rb]) {
                parent[ra] = rb;
            } else if (rank[ra] > rank[rb]) {
                parent[rb] = ra;
            } else {
                parent[rb] = ra;
                rank[ra]++;
            }
            return true;
        }
    }

    /**
     * 主入口。
     *
     * @param village   村边界。若非 null，会把“图斑覆盖外的残余面”也纳入切分，保证结果覆盖整个村域。
     * @param plots     原始图斑
     * @param threshold 单个分区允许包含的最大图斑数
     */
    public static List<Region> partition(Geometry village, List<? extends Geometry> plots, int threshold) {
        if (threshold <= 0) {
            throw new IllegalArgumentException("threshold must be > 0");
        }
        if (plots == null || plots.isEmpty()) {
            return Collections.emptyList();
        }

        GeometryFactory gf = plots.get(0).getFactory();
        List<Cell> cells = buildCells(village, plots, gf);
        List<Edge> edges = buildAdjacencyBySpatialIndex(cells);
        List<List<TreeEdge>> tree = buildMaximumSpanningTree(cells.size(), edges);

        int totalPlotCount = cells.stream().mapToInt(c -> c.plotCount).sum();

        List<List<Integer>> blocks = new ArrayList<>();
        List<Integer> allNodes = IntStream.range(0, cells.size()).boxed().collect(Collectors.toList());
        splitRecursively(allNodes, totalPlotCount, threshold, tree, cells, blocks);

        return buildRegions(blocks, cells, gf);
    }

    /**
     * 校验结果。
     */
    public static ValidationReport validate(
            Geometry village,
            List<? extends Geometry> originalPlots,
            List<Region> regions,
            int threshold) {

        ValidationReport report = new ValidationReport();
        if (regions == null || regions.isEmpty()) {
            report.errors.add("结果为空");
            return report;
        }

        // 1) 每块阈值校验 + 几何有效性
        for (Region region : regions) {
            if (region.plotCount > threshold) {
                report.errors.add("region " + region.regionId + " 超过阈值: " + region.plotCount + " > " + threshold);
            }
            if (region.geometry == null || region.geometry.isEmpty()) {
                report.errors.add("region " + region.regionId + " geometry 为空");
            } else if (!IsValidOp.isValid(region.geometry)) {
                report.errors.add("region " + region.regionId + " geometry 非法");
            }
        }

        // 2) 原始图斑必须且只能出现一次
        boolean[] seen = new boolean[originalPlots.size()];
        for (Region region : regions) {
            for (Integer plotId : region.plotIds) {
                if (plotId < 0 || plotId >= originalPlots.size()) {
                    report.errors.add("存在非法 plotId: " + plotId);
                    continue;
                }
                if (seen[plotId]) {
                    report.errors.add("plotId 重复分配: " + plotId);
                }
                seen[plotId] = true;
            }
        }
        for (int i = 0; i < seen.length; i++) {
            if (!seen[i]) {
                report.errors.add("plotId 未被分配: " + i);
            }
        }

        // 3) 分区之间不能有面积重叠
        STRtree regionIndex = new STRtree();
        for (Region region : regions) {
            regionIndex.insert(region.geometry.getEnvelopeInternal(), region);
        }
        for (int i = 0; i < regions.size(); i++) {
            Region a = regions.get(i);
            @SuppressWarnings("unchecked")
            List<Region> cands = regionIndex.query(a.geometry.getEnvelopeInternal());
            for (Region b : cands) {
                if (a.regionId >= b.regionId) continue;
                if (!a.geometry.getEnvelopeInternal().intersects(b.geometry.getEnvelopeInternal())) continue;
                double overlapArea = a.geometry.intersection(b.geometry).getArea();
                if (overlapArea > EPS) {
                    report.errors.add("region " + a.regionId + " 与 region " + b.regionId + " 存在重叠面积: " + overlapArea);
                }
            }
        }

        // 4) 若提供 village，则结果并集应覆盖村边界
        if (village != null) {
            List<Geometry> regionGeoms = regions.stream().map(r -> r.geometry).collect(Collectors.toList());
            Geometry union = unionCoverageOrFallback(regionGeoms, village.getFactory());
            double diffArea = village.symDifference(union).getArea();
            if (diffArea > EPS) {
                report.errors.add("分区并集与村边界不一致，symDifference area = " + diffArea);
            }
        }

        return report;
    }

    private static List<Cell> buildCells(Geometry village, List<? extends Geometry> plots, GeometryFactory gf) {
        List<Cell> cells = new ArrayList<>(plots.size() + 16);

        // 图斑：权重=1
        for (int i = 0; i < plots.size(); i++) {
            Geometry g = plots.get(i);
            if (g == null || g.isEmpty()) continue;
            cells.add(new Cell(cells.size(), g, 1, i));
        }

        // 若传入 village，则把未被图斑覆盖的残余面加入，权重=0，确保最终覆盖整个村边界
        if (village != null && !village.isEmpty()) {
            List<Geometry> plotGeoms = plots.stream()
                    .filter(Objects::nonNull)
                    .filter(g -> !g.isEmpty())
                    .collect(Collectors.toList());

            Geometry plotsUnion = unionCoverageOrFallback(plotGeoms, gf);
            Geometry residual = village.difference(plotsUnion);

            for (Polygon p : flattenPolygons(residual)) {
                if (p.getArea() > EPS) {
                    cells.add(new Cell(cells.size(), p, 0, -1));
                }
            }
        }

        return cells;
    }

    /**
     * 通用版邻接构建：
     * 用 STRtree 找候选对，再用共享边界长度判定。
     *
     * 若你的生产数据共边坐标完全一致，建议改成“边段哈希法”，复杂度会更好。
     */
    private static List<Edge> buildAdjacencyBySpatialIndex(List<Cell> cells) {
        STRtree index = new STRtree();
        for (Cell cell : cells) {
            index.insert(cell.geometry.getEnvelopeInternal(), cell.id);
        }

        Map<Long, Double> weightByPair = new HashMap<>();

        for (Cell a : cells) {
            @SuppressWarnings("unchecked")
            List<Integer> cands = index.query(a.geometry.getEnvelopeInternal());
            for (Integer bid : cands) {
                if (bid <= a.id) continue;
                Cell b = cells.get(bid);

                if (!a.geometry.getEnvelopeInternal().intersects(b.geometry.getEnvelopeInternal())) {
                    continue;
                }

                // 只认“共享边界长度 > 0”的邻接；只点接触不算
                double shared = a.geometry.getBoundary().intersection(b.geometry.getBoundary()).getLength();
                if (shared > EPS) {
                    long key = pairKey(a.id, b.id);
                    weightByPair.merge(key, shared, Double::sum);
                }
            }
        }

        List<Edge> edges = new ArrayList<>(weightByPair.size());
        for (Map.Entry<Long, Double> e : weightByPair.entrySet()) {
            int u = (int) (e.getKey() >>> 32);
            int v = (int) (long) e.getKey();
            edges.add(new Edge(u, v, e.getValue()));
        }
        return edges;
    }

    private static List<List<TreeEdge>> buildMaximumSpanningTree(int n, List<Edge> edges) {
        List<List<TreeEdge>> tree = new ArrayList<>(n);
        for (int i = 0; i < n; i++) {
            tree.add(new ArrayList<>());
        }

        if (n <= 1) {
            return tree;
        }

        edges.sort((a, b) -> Double.compare(b.sharedLength, a.sharedLength));

        DSU dsu = new DSU(n);
        int added = 0;
        for (Edge e : edges) {
            if (dsu.union(e.u, e.v)) {
                tree.get(e.u).add(new TreeEdge(e.v, e.sharedLength));
                tree.get(e.v).add(new TreeEdge(e.u, e.sharedLength));
                added++;
                if (added == n - 1) break;
            }
        }

        if (added != n - 1) {
            throw new IllegalStateException("邻接图不是连通的，无法构建生成树。请检查："
                    + "1) 图斑是否存在间隙；2) 是否只有点接触；3) 是否需要把残余面纳入建模。");
        }

        return tree;
    }

    private static void splitRecursively(
            List<Integer> nodes,
            int plotWeight,
            int threshold,
            List<List<TreeEdge>> tree,
            List<Cell> cells,
            List<List<Integer>> outBlocks) {

        if (plotWeight <= threshold) {
            outBlocks.add(nodes);
            return;
        }

        BlockSplit split = splitCurrentBlock(nodes, plotWeight, threshold, tree, cells);
        if (split == null) {
            // 理论上不该常见；若出现，先保底输出，后续可替换更强切分器
            outBlocks.add(nodes);
            return;
        }

        splitRecursively(split.leftNodes, split.leftWeight, threshold, tree, cells, outBlocks);
        splitRecursively(split.rightNodes, split.rightWeight, threshold, tree, cells, outBlocks);
    }

    /**
     * 在当前块对应的树诱导子图中，找一条树边切开。
     * 选择标准：
     * 1) 优先保证未来仍可在最少块数下继续拆到 <= threshold
     * 2) 尽量接近目标平衡
     * 3) 尽量切弱边（共享边界短）
     */
    private static BlockSplit splitCurrentBlock(
            List<Integer> nodes,
            int plotWeight,
            int threshold,
            List<List<TreeEdge>> tree,
            List<Cell> cells) {

        int m = nodes.size();
        if (m <= 1) {
            return null;
        }

        Map<Integer, Integer> pos = new HashMap<>(Math.max(16, (int) (m / 0.75f) + 1));
        for (int i = 0; i < m; i++) {
            pos.put(nodes.get(i), i);
        }

        int[] parentPos = new int[m];
        Arrays.fill(parentPos, -2);
        double[] parentEdgeWeight = new double[m];
        int[] order = new int[m];
        int ord = 0;

        int rootNode = nodes.get(0);
        int rootPos = 0;
        parentPos[rootPos] = -1;

        Deque<Integer> stack = new ArrayDeque<>();
        stack.push(rootNode);

        while (!stack.isEmpty()) {
            int uNode = stack.pop();
            int uPos = pos.get(uNode);
            order[ord++] = uPos;

            for (TreeEdge e : tree.get(uNode)) {
                Integer vPos = pos.get(e.to);
                if (vPos == null) continue;
                if (vPos == parentPos[uPos]) continue;
                if (parentPos[vPos] != -2) continue;

                parentPos[vPos] = uPos;
                parentEdgeWeight[vPos] = e.sharedLength;
                stack.push(e.to);
            }
        }

        if (ord != m) {
            throw new IllegalStateException("当前块在生成树上不是连通的，说明前面的切分逻辑有问题");
        }

        int[] subtreeWeight = new int[m];
        for (int i = 0; i < m; i++) {
            subtreeWeight[i] = cells.get(nodes.get(i)).plotCount;
        }
        for (int i = m - 1; i >= 0; i--) {
            int p = order[i];
            int pp = parentPos[p];
            if (pp >= 0) {
                subtreeWeight[pp] += subtreeWeight[p];
            }
        }

        int k = ceilDiv(plotWeight, threshold);
        if (k <= 1) {
            return null;
        }

        int kLeft = k / 2;
        int kRight = k - kLeft;
        double targetLeft = (double) plotWeight * kLeft / k;

        CandidateCut best = null;

        for (int p = 0; p < m; p++) {
            if (parentPos[p] < 0) continue; // root
            int s = subtreeWeight[p];
            int t = plotWeight - s;

            if (s == 0 || t == 0) continue;

            CandidateCut cand = evaluateCandidate(
                    s, t, kLeft, kRight, threshold, targetLeft, parentEdgeWeight[p], p);

            if (cand.betterThan(best)) {
                best = cand;
            }
        }

        if (best == null) {
            return null;
        }

        int cutChildNode = nodes.get(best.childPos);
        int cutParentNode = nodes.get(parentPos[best.childPos]);

        boolean[] inLeft = new boolean[m];
        Deque<int[]> dfs = new ArrayDeque<>();
        dfs.push(new int[]{cutChildNode, cutParentNode});

        while (!dfs.isEmpty()) {
            int[] cur = dfs.pop();
            int uNode = cur[0];
            int prevNode = cur[1];
            int uPos = pos.get(uNode);
            if (inLeft[uPos]) continue;
            inLeft[uPos] = true;

            for (TreeEdge e : tree.get(uNode)) {
                if (e.to == prevNode) continue;
                Integer vPos = pos.get(e.to);
                if (vPos == null) continue;
                if (!inLeft[vPos]) {
                    dfs.push(new int[]{e.to, uNode});
                }
            }
        }

        List<Integer> leftNodes = new ArrayList<>();
        List<Integer> rightNodes = new ArrayList<>();
        int leftWeight = 0;
        int rightWeight = 0;

        for (int i = 0; i < m; i++) {
            int node = nodes.get(i);
            if (inLeft[i]) {
                leftNodes.add(node);
                leftWeight += cells.get(node).plotCount;
            } else {
                rightNodes.add(node);
                rightWeight += cells.get(node).plotCount;
            }
        }

        if (leftNodes.isEmpty() || rightNodes.isEmpty()) {
            return null;
        }
        if (leftWeight == 0 || rightWeight == 0) {
            return null;
        }

        return new BlockSplit(leftNodes, rightNodes, leftWeight, rightWeight);
    }

    private static CandidateCut evaluateCandidate(
            int s,
            int t,
            int kLeft,
            int kRight,
            int threshold,
            double targetLeft,
            double cutEdgeWeight,
            int childPos) {

        boolean feasible = false;
        double imbalance = Double.POSITIVE_INFINITY;

        // 子树作为 left
        if (s <= kLeft * threshold && t <= kRight * threshold) {
            feasible = true;
            imbalance = Math.min(imbalance, Math.abs(s - targetLeft));
        }

        // 补集作为 left
        if (s <= kRight * threshold && t <= kLeft * threshold) {
            feasible = true;
            imbalance = Math.min(imbalance, Math.abs(t - targetLeft));
        }

        // 若没有严格可行解，退化为“尽量平衡”
        if (!feasible) {
            imbalance = Math.min(Math.abs(s - targetLeft), Math.abs(t - targetLeft));
        }

        return new CandidateCut(feasible, imbalance, cutEdgeWeight, childPos);
    }

    private static List<Region> buildRegions(List<List<Integer>> blocks, List<Cell> cells, GeometryFactory gf) {
        List<Region> regions = new ArrayList<>(blocks.size());

        for (int i = 0; i < blocks.size(); i++) {
            List<Integer> plotIds = new ArrayList<>();
            List<Geometry> geoms = new ArrayList<>();
            int plotCount = 0;

            for (Integer node : blocks.get(i)) {
                Cell cell = cells.get(node);
                geoms.add(cell.geometry);
                if (cell.originalPlotId >= 0) {
                    plotIds.add(cell.originalPlotId);
                    plotCount += 1;
                }
            }

            Geometry regionGeom = unionCoverageOrFallback(geoms, gf);
            regions.add(new Region(i, plotIds, regionGeom, plotCount));
        }

        return regions;
    }

    private static Geometry unionCoverageOrFallback(Collection<? extends Geometry> geoms, GeometryFactory gf) {
        if (geoms == null || geoms.isEmpty()) {
            return gf.createGeometryCollection();
        }

        Geometry[] arr = geoms.toArray(new Geometry[0]);
        Geometry gc = gf.createGeometryCollection(arr);

        try {
            // 对 valid coverage 更快
            return CoverageUnion.union(gc);
        } catch (Throwable ex) {
            // 脏数据或不是严格 coverage 时回退
            return UnaryUnionOp.union(Arrays.asList(arr), gf);
        }
    }

    private static List<Polygon> flattenPolygons(Geometry g) {
        List<Polygon> out = new ArrayList<>();
        collectPolygons(g, out);
        return out;
    }

    private static void collectPolygons(Geometry g, List<Polygon> out) {
        if (g == null || g.isEmpty()) return;
        if (g instanceof Polygon) {
            out.add((Polygon) g);
            return;
        }
        for (int i = 0; i < g.getNumGeometries(); i++) {
            collectPolygons(g.getGeometryN(i), out);
        }
    }

    private static long pairKey(int a, int b) {
        int x = Math.min(a, b);
        int y = Math.max(a, b);
        return (((long) x) << 32) | (y & 0xffffffffL);
    }

    private static int ceilDiv(int a, int b) {
        return (a + b - 1) / b;
    }

    // -------------------------

    // -------------------------

    public static void main(String[] args) {
        GeometryFactory gf = new GeometryFactory();

        List<Geometry> plots = buildGrid(gf, rows, cols, 0, 0, 1.0, 1.0);

        List<Region> regions = partition(village, plots, threshold);
        ValidationReport report = validate(village, plots, regions, threshold);

        System.out.println("region count = " + regions.size());
        System.out.println("valid = " + report.ok());
        if (!report.ok()) {
            System.out.println(report);
        }

        IntSummaryStatistics stat = regions.stream().mapToInt(r -> r.plotCount).summaryStatistics();
        System.out.println("min plots = " + stat.getMin());
        System.out.println("max plots = " + stat.getMax());
        System.out.println("avg plots = " + stat.getAverage());
    }

    private static List<Geometry> buildGrid(
            GeometryFactory gf,
            int rows,
            int cols,
            double x0,
            double y0,
            double cellW,
            double cellH) {

        List<Geometry> list = new ArrayList<>(rows * cols);
        for (int r = 0; r < rows; r++) {
            for (int c = 0; c < cols; c++) {
                double minX = x0 + c * cellW;
                double minY = y0 + r * cellH;
                list.add(rectangle(gf, minX, minY, minX + cellW, minY + cellH));
            }
        }
        return list;
    }

    private static Polygon rectangle(GeometryFactory gf, double minX, double minY, double maxX, double maxY) {
        Coordinate[] cs = new Coordinate[]{
                new Coordinate(minX, minY),
                new Coordinate(maxX, minY),
                new Coordinate(maxX, maxY),
                new Coordinate(minX, maxY),
                new Coordinate(minX, minY)
        };
        return gf.createPolygon(cs);
    }
```

## 6. 这版代码为什么这样写

这里我不只解释“是什么”，也解释“为什么”。

### 6.1 为什么先建图，而不是先切几何

因为“不能切图斑”是最高优先级约束。
只要分区是“图斑集合的并”，这个约束就天然成立。

这是一个非常重要的方法论：

**把硬约束前置到建模层，而不是放到求解后修补。**

这能让问题一下子从“几何切割 + 大量修补”变成“图划分 + 几何重建”。

### 6.2 为什么用共边长度做边权

因为你的图斑是地块，现实里大块共边通常意味着更强的空间连续性。
如果切掉一条很短的接触边，通常对子区块形状破坏更小；切掉一条很长的共享边，往往会让结果更奇怪。

所以边权用共边长度，是一种很自然的“切割代价”。

### 6.3 为什么用最大生成树，不直接做最小割

因为最小割本身只考虑 cut 小，不考虑 balance，也不天然保证未来还容易继续拆。JGraphT 的 `StoerWagnerMinimumCut` 确实提供了全局最小割实现，文档里给了复杂度说明，但它解决的是“cut 最小”，不是“连通平衡 k 分块”这个完整问题。它更适合用作局部 refinement 或小图二分子步骤，而不是直接拿来一把梭。([jgrapht.org][7])

最大生成树这一步其实是在做“降维”：

- 原图可能有很多环
- 树上切边一定得到两个连通块
- 所以连通性不用额外求证
- 复杂度和实现难度都明显下降

代价是它不是全局最优，但作为第一版基线非常合适。

### 6.4 为什么递归时按 `k = ceil(weight / T)` 来分配目标

这是为了避免一个常见坑：

比如当前块有 2500 个图斑，阈值 1000。
如果你总是硬劈成 1250 / 1250，后面会得到 4 块；
但其实最少块数是 3 块，理想大小应该接近 833 / 1667，然后再把 1667 继续拆成 833 / 834。

所以代码里不是简单追半，而是先估当前块最少需要几块，再把“未来块数”也纳入当前切分目标。这是从“局部平衡”走向“全局块数最优”的一个关键小技巧。

### 6.5 为什么 JTS 里优先用 `CoverageUnion`

因为你的输入如果真的是合法 coverage，那么 `CoverageUnion` 正好就是这个场景。JTS 文档明确说它对 valid polygonal coverage 更高效，但也明确提醒：它不会先帮你验证 coverage；如果输入不是合法 coverage，可能抛 `TopologyException` 或得到无效结果。所以上面的代码里用了 fallback：先尝试 `CoverageUnion`，失败后退回 `UnaryUnionOp`。([locationtech.github.io][5])

### 6.6 为什么还要提 `Polygonizer`

因为真实生产数据往往没你想得那么干净。
JTS 的 `Polygonizer` 适合在“线工作流”下诊断问题：它把线转成面，同时能报告 dangles、cut edges、invalid ring lines。文档也明确写了，它要求输入线是 correctly noded，也就是线只在端点相交。对你这种地块数据，如果以后发现“理论上应该是 coverage，但实际 union/切分老出错”，`Polygonizer` 是非常好的排错入口。([locationtech.github.io][8])

## 7. 复杂度怎么理解

先分两层。

### 7.1 如果按我这版代码

设：

- `n` = 单元数（图斑 + 残余面）
- `m` = 邻接边数

构图部分：

- STRtree 建索引和查询，典型情况下是高效的，文档说明它是 packed R-tree。([locationtech.github.io][4])
- 但要注意，**最坏情况** 仍可能退化，因为如果很多包围盒互相重叠，候选对会很多。

所以这版构图复杂度更准确地说是：

`O(n log n + candidatePairs * geometryCheckCost)`

在地块型数据里，通常比 `O(n^2)` 好得多，但它不是最理想的生产构图方案。

最大生成树：

`O(m log m)`

递归切树：

每次只在当前树块上做 DFS 和一次候选扫描。
如果切分比较平衡，整体通常接近 `O(n log k)`；
极端不平衡时会差一些，但仍远比做全局 ILP 友好。

空间复杂度：

`O(n + m)`

### 7.2 生产优化版应该怎么想

如果你的数据真的满足“共边坐标完全一致、拓扑很干净”，那最值得优化的不是后面的树切分，而是前面的**邻接构图**。

生产版我更建议你把 `buildAdjacencyBySpatialIndex` 换成：

**边段哈希法**

做法是：

1. 遍历每个图斑的所有边段
2. 把边段方向规范化（A-B 和 B-A 视为同一条）
3. 用边段 key 做哈希
4. 同一条边段出现两次，就说明这两个图斑相邻
5. 累加边长得到共边长度

如果数据真干净，这样的构图复杂度更接近：

`O(S)`

其中 `S` 是总边段数。

这通常会比“STRtree 找候选 + 边界相交计算”更快，也更稳。这一点非常值得你自己再推一遍，因为它体现了一个很重要的工程思维：

**当业务数据已经自带拓扑结构时，不要再拿通用几何算法硬算一遍。**

## 8. 测试与验证方案

你问的是算法，不只是代码，所以测试必须分层。

### 8.1 先做合成数据测试

建议至少做 5 类。

第一类，规则网格。
比如 `100 x 100 = 10000` 个矩形图斑，阈值 1000。
这是最基础的正确性和均衡性测试。

第二类，长条村。
村边界极细长，看是否切出极差形状。

第三类，环形 / 带洞村。
检验残余面、洞、边界重建是否稳。

第四类，密度不均匀。
一半区域图斑很密，一半区域很稀。
检验“按图斑数平衡”是否导致面积极不平衡，这本身不一定是 bug，而是目标函数选择的结果。

第五类，脏数据近似场景。
故意制造：

- 只有点接触
- 细缝
- 微小重叠
- 微小未覆盖残片

看验证器能否尽快报出来。

### 8.2 验证指标

不要只看“跑没跑出来”，至少看这几个指标：

1. `blockCount`：最终块数，是否等于或接近 `ceil(N / T)`
2. `maxPlotCount`：是否都 `<= T`
3. `min/max/stddev`：均衡性
4. `cutLength`：切断共边总长度
5. `compactness`：例如 `perimeter^2 / area`
6. `coverageDiffArea`：与原村边界的 `symDifference` 面积
7. `validity`：每个 region 是否 valid
8. `overlapArea`：分区两两重叠面积是否为 0

### 8.3 小规模精确对照

再做一件很有学习价值的事：

拿一个很小的数据集，比如 50~200 个图斑，
把它 coarsen 后用 ILP/CP-SAT 求一个“尽量精确”的答案，
再和启发式结果对比：

- 块数差多少
- imbalance 差多少
- cutLength 差多少

这样你就能知道自己的启发式到底“差在哪”，而不是只知道“能跑”。

## 9. 如果追求更高质量，下一步该怎么升级

### 9.1 升级路线一：树切分后做局部 refinement

这是最自然的一步。

思路是：

- 先用上面这版得到一个可行解
- 找边界上的图斑
- 尝试把某个边界图斑从 A 挪到 B
- 前提：
  - A 去掉它后仍连通
  - 它加入 B 后 B 仍连通
  - 不超过阈值

- 如果 cutLength 更小、均衡更好，就接受

这一步很像图划分里的局部搜索。

### 9.2 升级路线二：用 KaHIP 做高质量 connected partition

如果你接受外部二进制而不是纯 Java，我会更推荐这个方向。
KaHIP 当前 README 里明确写了它做高质量平衡图划分，并且 v3.25 新增了实验性的 `--connected_blocks` 来产出每块都连通的结果。([GitHub][3])

你的流程会变成：

1. Java/JTS 构建图斑邻接图
2. 导出为 METIS/KaHIP 图格式
3. 调 KaHIP 做 `k = ceil(N / T)` 划分
4. 把 block id 回写到图斑
5. 对每个 block 做 `CoverageUnion`

这通常会比纯树切分质量更高，尤其在复杂形状上。

### 9.3 升级路线三：coarsen + 精确求解

如果你特别在意“尽量接近真正最优”，那可以：

1. 先把很多紧密相连的图斑聚成超点
2. 在超点图上做 ILP / CP-SAT
3. 最后再局部展开 refinement

这条路线比较像工业优化器的套路，但实现复杂度明显更高。

## 10. 这个例子你真正该内化的东西

我建议你把这道题当成一个“算法建模训练题”，而不只是一个 GIS 题。

你应该带走这几个方法论。

第一，先分清**硬约束**和**优化目标**。
“不能切图斑”是硬约束；“尽量均匀”是优化目标。
硬约束应该尽量通过建模天然满足，而不是事后补救。

第二，遇到几何问题，先问自己：
**这其实是不是图问题？**

你的题就是典型例子。
表面上是切多边形，实质上是平面图连通平衡划分。

第三，不要轻易说“最优”。
没有明确目标函数，“最优”是空话。
只有阈值约束时，问题甚至会退化。

第四，先做“正确且可解释”的基线，再追求高质量。
树切分不是最强，但它特别适合第一版，因为：

- 易实现
- 易验证
- 易解释
- 易扩展

第五，复杂度分析不要只写大 O。
要问：

- 真正的瓶颈在哪
- 通用算法为何慢
- 业务数据有什么结构能利用

比如这题里，最值得利用的结构就是“共边很多，而且拓扑干净”。

## 11. 可参考的书、论文和开源方案

如果你要系统学，我建议按这个顺序看。

先看基础：

《Computational Geometry: Algorithms and Applications》。这是计算几何经典教材，Springer 页面也明确把它定位为 computational geometry 的广泛综述教材。你不一定全看，但里面关于 polygon、range searching、BSP、quadtrees 的思路对你很有帮助。([Springer][9])

再看图划分：

《Graph Partitioning》这本书是图划分专题书；Wiley/ISTE 的介绍也明确说它总结了 graph partitioning 的理论基础与主要应用。([Wiley Online Library][10])

再看谱方法：

Fan Chung 的《Spectral Graph Theory》。如果你之后想理解 spectral bisection、normalized cut，这本非常值得。AMS 和作者页面都有介绍。([bookstore.ams.org][11])

再看 connected balanced partition 的理论难度和近似：

Moura, Ota, Wakabayashi 2023 这篇论文很适合直接对应你这个问题，因为它讨论的就是 balanced connected partitions，并给出 hardness、approximation、parameterization 等结果。([数学、统计与计算机科学院][1])

开源和工程工具：

- JTS：Java 侧几何基础库，尤其是 `STRtree`、`CoverageUnion`、`Polygonizer`、`IsValidOp`。([locationtech.github.io][4])
- METIS：经典多层图划分框架，适合理解 multilevel recursive bisection / k-way。([GitHub][12])
- KaHIP：现在更值得你关注，尤其是 connected blocks。([GitHub][3])
- CGAL `Partition_2`：不是你这题的直接解，但适合帮助你区分“经典 polygon partition”和“图斑不可切分组”这两类问题。([CGAL][2])

如果你愿意，我下一轮可以继续两件事里的其中一件：
一是把上面这版纯 Java 再升级成“树切分 + 边界局部优化”的第二版；二是直接给你补一版“生产级方案设计”，专门讨论 10 万级数据下如何把邻接构图从几何相交改成边段哈希。
