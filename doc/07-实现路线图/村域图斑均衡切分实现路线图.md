# 村域图斑均衡切分实现路线图

## 1. 实际业务问题

输入是一批农田地块 Shapefile。每个图斑带有行政属性，其中 `CUNDM` 表示村编码。目标是按村生成任务包：

- 同一个任务包内的图斑数量不超过阈值，例如 1000。
- 同一个任务包内的图斑应尽量空间连续。
- 同一个村拆出的多个任务包大小应尽量均衡，避免一个包很大、另一个包很小。

这个问题不是单纯的文件读取问题，也不是单纯的几何问题。它的核心是：先把空间接触关系转换成图，再在图上做连通且均衡的切分。

## 2. 当前项目位置

当前代码已经完成第一阶段基础设施：

- `io.ShapefileReader`：读取 Shapefile，输出 GeoTools `SimpleFeature`。
- `mapper.PlotMapper` / `mapper.SichuanPlotMapper`：把外部字段映射为内部 `domain.Plot`。
- `geometry.GeometryValidator` / `geometry.QCReport`：检查无效几何并输出 QC 报告。
- `geometry.NeighborIndex` / `geometry.GeometryOps`：用 STRtree 找候选邻居，并计算共享边界长度。
- `graph.AdjacencyGraphBuilder`：把图斑集合转换成带权邻接图。
- `graph.MaxSpanningTree`：用 Kruskal 计算 Maximum Spanning Tree。
- `cut.TreeCutStrategy` / `cut.TargetedPeelingStrategy` / `cut.BalancedCutStrategy`：切点选择策略接口与两种实现。
- `cut.algorithm.RecursivePeelingAlgorithm`：递归剥离算法，配合 `TreeCutStrategy` 驱动切分，主策略为 `TargetedPeelingStrategy`。
- `cut.algorithm.GlobalWeakestEdgeAlgorithm`：切 k-1 条最弱边，适用于已知分组数场景。
- `cut.model.PartitionContext` / `cut.model.TreeCutDecision`：切分参数与决策结果的值对象。

下一阶段要实现的是 Task B 集成层：`VillagePartitioner`，把 Plot 列表接入完整切分链路。

## 3. 总体处理链路

完整链路如下：

```text
Shapefile
  -> SimpleFeature 列表
  -> Plot 列表
  -> Geometry QC
  -> 按 CUNDM 分村
  -> 每个村构建空间邻接图
  -> 按连通分量拆分
  -> 每个连通分量构建 Maximum Spanning Tree
  -> 在树上递归做均衡切分
  -> 输出任务包
```

这条链路把复杂的空间切分问题分成三层：

1. 数据适配层：把外部 Shapefile 和字段名隔离在 `io`、`mapper` 中。
2. 空间图构建层：把几何相邻关系转换成带权边。
3. 图切分层：在图或树上做连通均衡划分。

## 4. 为什么采用 Maximum Spanning Tree

直接在一般图上做最优连通均衡划分，问题复杂且实现成本高。项目采用 Maximum Spanning Tree 作为工程化降维：

- 原始邻接图可能有大量边，切分搜索空间很大。
- Maximum Spanning Tree 保留了连接强度最大的骨架边。
- 树结构上切一条边一定得到两个连通子树，连通性容易保证。
- 树上的递归切分更容易解释、测试和调试。

代价是：MST 是近似模型，不保证全局最优。但它保留强邻接关系，行为透明，适合作为第一版稳定算法。

## 5. 关键算法步骤与复杂度

下面以一个村内 `m` 个图斑、邻接图 `E` 条边为单位说明。

### 5.1 构建空间索引

使用 JTS STRtree 对图斑 envelope 建索引。

- 时间复杂度：通常接近 `O(m log m)`。
- 空间复杂度：`O(m)`。
- 作用：避免所有图斑两两比较。

如果不用空间索引，候选邻居检查是 `O(m^2)`，真实数据上成本过高。

### 5.2 查询候选邻居并计算共享边界

对每个图斑，用 envelope 查询候选几何，再用 `GeometryOps.sharedBoundaryLength(a, b)` 判断是否共享边界。

- 平均情况：接近 `O(m log m + E * g)`。
- 最坏情况：如果大量 envelope 重叠，可能退化到 `O(m^2 * g)`。
- `g` 表示单次几何边界计算的成本，和几何复杂度有关。

输出是带权无向图：节点是图斑下标，边权是共享边界长度。

### 5.3 连通分量拆分

真实村域可能在空间上不连续。必须先对邻接图做 connected components。

- 时间复杂度：`O(m + E)`。
- 空间复杂度：`O(m + E)`。

每个连通分量独立切分。孤立单点分量直接形成一个任务包。

### 5.4 Maximum Spanning Tree

对每个连通分量运行 Kruskal，按边权从大到小选边。

- 时间复杂度：`O(Ec log Ec)`，`Ec` 是该分量内边数。
- 空间复杂度：`O(Vc + Ec)`。

结果是一棵树，包含 `Vc - 1` 条边。

### 5.5 递归剥离切树（已实现）

采用 `RecursivePeelingAlgorithm` + `TargetedPeelingStrategy` 组合，每次从树边缘剥离一个大小尽量接近 `maxGroupSize` 的子树：

```text
如果当前树大小 <= maxGroupSize：
  返回当前组
否则：
  DFS 计算每个节点的 subtree size
  找最大且不超过 maxGroupSize 的子树（TargetedPeelingStrategy）
    → 无合格候选时退而选最小的超标子树
  切出该子树作为一组
  剩余部分递归处理
```

与 50/50 均衡切不同：
- 50/50 切（`BalancedCutStrategy`）每次减半，递归深度 `O(log K)`，但容易在接近阈值时过度切分。
- 目标尺寸剥离（`TargetedPeelingStrategy`）每次切出一个合格大小的组，避免过切，组间差更小。

单次剥离成本：

- DFS 计算 subtree size：`O(Vc)`。
- 遍历找最佳切点：`O(Vc)`。
- 收集子树节点：`O(Vc)`。

总复杂度：`O(Vc² / maxGroupSize)`，即切分次数 × 单次成本。

## 6. 后续实现路线

### ✅ 阶段 1：仓库里程碑整理（已完成）

- 包结构清晰：domain / io / mapper / geometry / graph / cut / cut.algorithm / cut.model。
- QC 链路完整，smoke test 验证 65,387 个图斑。
- 所有测试通过，RealDataSmokeTest 默认 `@Disabled`。

### ✅ 阶段 2：树级切分（已完成）

- `TreeCutStrategy` 接口 + `TargetedPeelingStrategy` / `BalancedCutStrategy` 两种实现。
- `RecursivePeelingAlgorithm`：递归剥离，每次切出接近 `maxGroupSize` 的组。
- `GlobalWeakestEdgeAlgorithm`：按目标分组数切 k-1 条最弱边。
- 全部通过链式图、星形图、平衡二叉树等场景测试。

### 🔲 阶段 3：村级切分编排（当前下一步）

目标：把 Plot 输入接入完整图切分链路。

- 新增 `cut.VillagePartitioner`。
- 输入：`List<Plot> villagePlots`、`maxGroupSize`。
- 内部提取 `Plot.geometry()` 调用现有图构建能力。
- 按连通分量拆分：分量 > maxGroupSize → MST + RecursivePeelingAlgorithm；分量 ≤ maxGroupSize → 暂存。
- **小分量合并**：实测均值约 23 图斑/分量，远小于 maxGroupSize，必须贪心装箱合并后再成组。
- 输出：`List<List<Plot>>`。
- 验收不变量：每个 Plot 出现且仅出现一次（无遗漏无重复）。

### 🔲 阶段 4：真实数据验证

目标：用四川测试数据确认工程可运行，并输出可解释统计。

- 按村统计输入图斑数量。
- 按村统计输出任务包数量、最大包、最小包、平均包。
- 统计空间连通分量数量。
- 对无效几何继续通过 QC 报告暴露，不在图构建阶段静默修复。

### 🔲 阶段 5：输出与可视化

目标：让结果能被业务侧检查。

- 输出任务包编号与图斑 id/referenceId 的映射。
- 后续可考虑写 Shapefile/GeoJSON，把任务包编号作为属性写回。
- 可视化不是核心算法前置条件，应在算法稳定后再做。

## 7. 重要约束

- `SimpleFeature` 不进入核心算法，避免 GeoTools 类型污染 domain 和 graph。
- 几何修复不在邻接图构建阶段自动进行，避免隐藏数据质量问题。
- `study` 包只作为学习材料，不作为生产依赖。
- 分组结果是集合划分：每个 Plot 出现且仅出现一次，地理边界不重合由此自动保证，无需单独校验几何。
