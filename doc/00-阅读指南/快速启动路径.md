# 快速启动路径

这份路径只服务一个目标：

尽快把你带到“可以自己开始写代码”的状态。

不是先学完全部理论，而是先补最小基础，再进入第一版实现。

## 第 0 步：先把问题模型看对

先读：

`../01-问题建模与最优标准/村域图斑重分组：邻接图上的连通平衡划分.md`

读到这几个结论为止：

- 这不是普通几何切割问题
- 图斑是不可再分的原子单元
- 真正的模型是“邻接图上的连通平衡划分”
- “最优”必须先定义硬约束和软目标

如果这一步没立住，后面所有代码都容易写偏。

## 第 1 步：补最小图论

再读：

`../03-图论基础与最小练习/图斑重分组的图论入门：从业务映射到最小知识树.md`

你的目标不是记定义，而是能把下面这组映射顺口说出来：

- 图斑 = 节点
- 共边 = 边
- 一组图斑是不是一整块 = 这组节点是否连通
- 一次切分 = 把图分成若干子图
- 每块图斑数不超过阈值 = 每个子图的节点数不超过阈值

## 第 2 步：马上开始写第一段代码

读：

`../03-图论基础与最小练习/图论入门第一课：用Java实现连通分量.md`

然后自己写下面四个东西：

1. `Graph`
2. `addUndirectedEdge(int u, int v)`
3. `connectedComponents(Graph graph)`
4. `isConnectedSubset(Graph graph, Set<Integer> subset)`

这一阶段你先完全不要碰 JTS，不要碰 KD 树，也不要碰村边界。

只做纯图。

## 第 3 步：开始补树和最大生成树

接着读：

`../06-最大生成树方案专项/最大生成树学习路径与分析/最大生成树切分方案：深度分析.md`

以及：

`../06-最大生成树方案专项/最大生成树学习路径与分析/最大生成树学习路径可视化.jsx`

这一步你要补的不是全部算法，只补四件事：

1. `UnionFind`
2. 最大生成树
3. 子树大小
4. 切树

自己写的顺序建议是：

1. `UnionFind`
2. `kruskalMaxST(...)`
3. `computeSubtreeSizes(...)`
4. `findBestCutEdge(...)`

## 第 4 步：再回头看空间基准法

再读：

`../02-空间基准方案/空间划分基准：从KD树、四叉树到连通性修复.md`

`../02-空间基准方案/KD树图斑分组基准：纯空间切分与连通拆分.md`

这时候你会更容易看懂：

- 为什么空间法适合做基准
- 为什么它不天然保证连通
- 为什么它最终还是会往图论修复靠近

## 第 5 步：再看完整 Java 工程

最后再读：

`../04-Java实现与示例工程/村域图斑空间分区方案与Java示例/村域图斑空间分区方案：思路总览与实现说明.md`

然后进：

`../04-Java实现与示例工程/村域图斑空间分区方案与Java示例/示例代码/`

这一阶段你不是第一次看代码，而是带着问题去看：

- 这个类负责什么
- 哪一部分是空间索引
- 哪一部分是邻接构图
- 哪一部分是切分
- 哪一部分是验证

## 一周启动建议

### Day 1

读“村域图斑重分组：邻接图上的连通平衡划分”

输出你自己的 5 句话版本问题定义。

### Day 2

读“图斑重分组的图论入门：从业务映射到最小知识树”

自己画出“图斑 -> 图”的映射图。

### Day 3

读“图论入门第一课：用Java实现连通分量”

写出 `Graph + connectedComponents`。

### Day 4

自己写 `isConnectedSubset`，并手工造小图测试。

### Day 5

开始写 `UnionFind` 和最大生成树。

### Day 6

写“树上切边”的最小玩具版本。

### Day 7

回看“空间基准方案、图论基础与最小练习、最大生成树方案专项”，把你的理解和现有材料对齐。

## 你的真正起点

如果你想今天就开始，不要从复杂工程开始。

今天最该做的是：

1. 读“村域图斑重分组：邻接图上的连通平衡划分”
2. 读“图斑重分组的图论入门：从业务映射到最小知识树”
3. 按“图论入门第一课：用Java实现连通分量”自己写出 `connectedComponents`

做到这一步，你就已经从“在看资料”进入“开始掌握算法”了。
