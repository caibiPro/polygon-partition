# 最大生成树切分方案：深度分析

---

## 为什么说这是最优雅的方案？

三种方案的"优雅度"可以从一个角度衡量：**用了多少额外的工程来保证约束**。

| 约束 | 坐标递归二分 | METIS图划分 | 最大生成树 |
|------|-------------|------------|-----------|
| 拓扑安全 | 需要投影间隙检测 | 天然保证 | 天然保证 |
| 连通性 | **不保证** | **不保证** | **天然保证** |
| 均衡性 | 中位数近似 | KL/FM迭代优化 | 子树大小精确计算 |
| 几何恢复 | 半平面求交（稳健） | Union+间隙分配（复杂） | Union（简单，因为连通） |

**连通性是关键差异点。** 前两种方案都可能产生不连通的区域（比如L形区域被直线切分后，一侧的图斑在空间上分成两块）。最大生成树方案中，删除一条树边后的两个分量，**数学上保证是连通的**。

---

## 核心算法逐步拆解

### Step 1: 邻接图

```
输入: N个图斑多边形

    ┌───┬───┬───┐
    │ A │ B │ C │    每对共边的图斑之间建一条边
    ├───┼───┼───┤    边权 = 共边长度
    │ D │ E │ F │
    ├───┼───┼───┤    为什么用共边长度做权重？
    │ G │ H │ I │    → 共边越长，两个图斑关系越密切
    └───┴───┴───┘    → 权重体现了"不应该被切开"的强度

邻接图:
    A—8—B—5—C
    |   |   |
    4   9   6        数字 = 共边长度
    |   |   |
    D—7—E—10—F
    |   |   |
    3   8   5
    |   |   |
    G—6—H—7—I
```

**技术细节**：用STR-tree做空间索引，避免O(N²)两两比较。每个图斑查询邻居只需O(log N + k)。

### Step 2: 最大生成树（Kruskal算法）

```
边按权重降序排列: E-F(10), B-E(9), ...

贪心选择（不构成环就选）：

Round 1: E—10—F  ✓ 选入（最重边）
Round 2: B—9—E   ✓ 选入
Round 3: A—8—B   ✓ 选入
Round 4: E—8—H   ✓ 选入
Round 5: D—7—E   ✓ 选入
Round 6: H—7—I   ✓ 选入
Round 7: C—6—F   ✓ 选入
Round 8: G—6—H   ✓ 选入
Done!  8条边 = 9个节点 - 1 ✓

最大生成树:
    A—8—B   C
    |   |   |
    ×   9   6     × = 被丢弃的边
    |   |   |
    D—7—E—10—F
        |
        8         注意: 树保留了最强的连接
        |
    G—6—H—7—I
```

**为什么是最大而不是最小？**

这个问题值得反复咀嚼：

```
我们的目标: 切割时从"最弱的地方"切开

最小生成树保留了: 1, 2, 3, 4, 5, 5, 6, 6 (最小的N-1条边)
  → 树上最强的边权 = 6
  → 切割时可能切掉权重6的边 → 把共边长度为6的相邻图斑分开了

最大生成树保留了: 10, 9, 8, 8, 7, 7, 6, 6 (最大的N-1条边)  
  → 树上最弱的边权 = 6
  → 切割时优先切权重6的边 → 只从共边最短的地方分开
  
差异: 最大ST保证了"被切断的连接一定是所有保留连接中最弱的"
```

### Step 3: 递归平衡切树

```
树:     A—8—B   C
        |   |   |
        ×   9   6
        |   |   |
        D—7—E—10—F
            |
            8
            |
        G—6—H—7—I

假设阈值K=3，需要切成 ⌈9/3⌉ = 3 个区域。

第一刀: 遍历每条树边，计算删除后两侧大小

  删A-B: [A](1) vs [B,C,D,E,F,G,H,I](8)     差=7
  删B-E: [A,B](2) vs [C,D,E,F,G,H,I](7)     差=5
  删C-F: [C](1) vs [A,B,D,E,F,G,H,I](8)     差=7
  删D-E: [D](1) vs [A,B,C,E,F,G,H,I](8)     差=7
  删E-F: [A,B,D,E](4) vs [C,F,G,H,I](5)     差=1 ← 最均衡！
  删E-H: [A,B,C,D,E,F](6) vs [G,H,I](3)     差=3
  删G-H: [G](1) vs [A,B,C,D,E,F,H,I](8)     差=7
  删H-I: [I](1) vs [A,B,C,D,E,F,G,H](8)     差=7

→ 选E-F(权重10)，差=1...但权重是10，这是最强连接
→ 选E-H(权重8)，差=3，比较均衡而且权重较小

实际策略: 先按均衡度排序，均衡度相同时选权重最小的边。

假设选E-H:  
  左: {A,B,C,D,E,F} = 6个图斑
  右: {G,H,I} = 3个 ≤ K ✓ 停止

第二刀: 对左侧{A,B,C,D,E,F}继续
  类似分析...选B-E或E-F

最终结果:
  区域1: {A,B} 或 {A,B,D}
  区域2: {C,D,E,F} 或 {C,E,F}  
  区域3: {G,H,I}

每个区域内的图斑保证连通 ← 这是树的核心性质!
```

## 时间复杂度精确分析

```
Step 1: 建邻接图
  STR-tree构建:     O(N log N)
  邻居查询:         O(N × k × T_inter)
                    k≈8 (平均邻居数), T_inter=JTS intersection时间
  总计:             O(N log N) + O(N) ≈ O(N log N)

Step 2: 最大生成树  
  排序E条边:         O(E log E), E≈4N~8N → O(N log N)
  并查集操作:        O(E × α(N)) ≈ O(E) ≈ O(N)
  总计:             O(N log N)

Step 3: 递归切树
  每次切割:          O(M) 计算子树大小 + O(M) 遍历边, M=当前子树大小
  递归深度:          O(log(N/K))
  每层总工作量:       O(N) (所有子树大小之和)
  总计:             O(N log(N/K))

总计: O(N log N)   ← 瓶颈在Step 1和Step 2
```

对比:
```
坐标递归二分:   O(N log N)        ← 瓶颈在排序
METIS图划分:    O(N log N)        ← 瓶颈在建图
最大生成树:     O(N log N)        ← 瓶颈在建图+排序

三种方案渐近复杂度相同！
但常数因子不同：
  坐标二分 < 最大生成树 < METIS图划分
```

## 与前两种方案的关键对比

### 对比坐标递归二分

```
场景: L形区域中的图斑

    ┌─────────┐
    │ A B C D │
    ├────┐    │
    │ E F│    │    坐标二分: 垂直线切割 → F和G被分到不同侧
    │    │    │               但E和H变成了不连通的碎片
    │ G H│    │
    └────┘    │    最大生成树: 沿图斑邻接关系切割
              │               E-F-G-H作为连通子树不会被打断
              │
```

**坐标二分的盲区**: 它不知道图斑之间的连接关系，纯粹按坐标切割。对于不规则形状的区域，容易产生不连通的子区域。

**最大生成树的优势**: 切割发生在树边上，而树本身编码了连接关系。删除任意一条树边后，两侧一定是连通的。

### 对比METIS图划分

```
METIS: 粗化→划分→细化
  优点: 全局最优化切边数
  缺点: 不保证连通性（需要后处理检查）
        几何恢复复杂（Union后可能有多个不连通块）
        实现复杂（多层级+KL/FM+粗化策略）

最大生成树: 建树→切树
  优点: 连通性天然保证
        实现简单（Kruskal + DFS）
        切割语义明确
  缺点: 不是全局最优的切边方案
        （因为生成树丢掉了一些边的信息）
```

**核心trade-off**: 最大生成树用"全局最优性"换取了"连通性保证+实现简洁"。在实际GIS应用中，连通性通常比切边最优更重要（不连通的区域在业务上没有意义）。

## 注意事项与边界情况

### 1. 孤立图斑

如果某些图斑与其他图斑完全不相邻（没有共边），它们在邻接图中是孤立节点，不会出现在生成树中。

**处理**: 对不连通分量分别建树、分别切割，最后合并结果。

### 2. 阈值为1

当K=1时，每个图斑单独成一个区域。此时不需要建树，直接每个图斑一组。

### 3. 图斑数远大于阈值

当N/K很大时（如10万图斑、阈值100），递归深度约为log₂(1000)≈10层，非常可控。

### 4. 并查集的路径压缩

并查集的find操作使用路径压缩（path compression），使得后续查询近O(1)。这是Kruskal算法能在O(E log E)内完成的关键。

```java
int find(int x) {
    if (parent[x] != x) 
        parent[x] = find(parent[x]); // 路径压缩：直接指向根
    return parent[x];
}
```

### 5. 切割边的选择策略可以调优

当前策略: 均衡度优先，权重次之。

可选策略:
- 纯均衡: 只看两侧大小差
- 权重优先: 先切权重最小的边（从最弱处切）
- 加权综合: score = α × imbalance + β × weight
- 带阈值约束: 确保两侧都 ≤ threshold

不同策略适合不同业务需求。
