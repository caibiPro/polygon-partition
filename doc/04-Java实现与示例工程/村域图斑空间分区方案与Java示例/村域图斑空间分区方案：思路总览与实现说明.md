## 思维路径：从简单到复杂

**一维**：N个线段排在一条线上，排序后在间隙处切一刀，问题几乎trivial。核心约束浮现：**只能在间隙处切，不能穿过已有元素**。

**二维网格**：沿网格线切割，本质是一维的两个方向叠加。递归二分的雏形出现。

**二维不规则（本题）**：核心困难在于——切割线如何在不规则图斑之间的"缝隙"中穿行？

**关键转化**：将几何问题转化为图论问题。图斑→节点，共边→边，切割线→切边。这就是整个方案的思维核心。

## 四种方案对比

我给出了四种方案：

1. **坐标递归二分**——简单直接，按质心排序从中间切
2. **图划分（METIS风格）**——理论最优但实现复杂，从图划分恢复几何区域是难点
3. **Voronoi区域生长**——直觉上自然但均衡性难控制
4. **混合方案（推荐）**——坐标二分 + 图斑感知切割，兼顾简洁和正确性

## 代码结构

提供了两套完整Java实现：

`SpatialBisectionVillageSplitter.java` 是推荐的混合方案，核心逻辑：递归选择较长轴方向→质心排序→中位数附近搜索安全切分位置→半平面求交切割多边形→递归。内置了完整的验证器和评估指标（均衡度、紧凑度等）。

`GraphPartitionVillageSplitter.java` 是图划分方案的对比实现，含邻接图构建、KL细化算法。

`VillagePartitionDemoRunner.java` 提供5组测试场景（网格、不规则、边界情况、WKT可视化输出、性能基准），直接输出WKT可以粘贴到QGIS里查看效果。

## 需要你关注的几个"为什么"

学习笔记中详细解释了每个关键选择的原因，这里点几个最重要的：

**为什么用STR-tree而不是普通R-tree**：我们的场景是"所有数据已知，批量查询"，STR-tree的排序-分块-递归构建策略让包围盒重叠极少，查询效率更高。

**为什么坐标排序能近似谱方法**：对于空间嵌入图，质心坐标和拉普拉斯矩阵的Fiedler向量高度相关——空间上近的节点在图上也连通。

**为什么用半平面求交而不是直接split**：JTS没有原生的"用线切多边形"操作，但 `polygon.intersection(halfPlane)` 是稳健的标准操作，效果等价。
