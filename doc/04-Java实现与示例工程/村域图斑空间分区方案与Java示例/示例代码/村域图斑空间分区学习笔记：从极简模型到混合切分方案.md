# 村域图斑空间分区算法：完整学习笔记

## 第一部分：从极简模型到完整方案的思维推导

### 1.1 一维情形：线段上的分组

**问题简化**：N个不重叠的小线段分布在一条大线段[0, L]上，将大线段切分为若干子段，每段包含 ≤ K 个小线段。

**解法**：将小线段按位置排序，从左到右每K个一组，在相邻组之间的空隙处切一刀。

**复杂度**：O(N log N) 排序 + O(N/K) 次切分。

**关键洞察**：
- 排序后，切分点的选择只需在两个相邻线段之间的"间隙"中选一个点
- 切分线不能穿过任何小线段 → 只能在间隙中切
- 这就是后面所有方案的核心约束

### 1.2 二维网格情形：规整格网

**问题简化**：图斑排列在规则网格上，每个格子一个图斑。

**解法**：沿网格线切割，本质上就是二维数组的分块问题。先沿X轴每√K列切一刀，再沿Y轴每√K行切一刀。

**关键洞察**：
- 网格线天然就是"不穿过图斑的切割线"
- 二维需要两个方向的递归切分（递归二分法的雏形）

### 1.3 二维不规则情形（本题）

从网格过渡到不规则多边形，核心困难出现了：

1. **切割线的几何约束**：不能穿过任何图斑，必须在图斑之间的"缝隙"中穿行
2. **连通性约束**：切分后的每个子区域必须是拓扑合法的多边形
3. **均衡性目标**：各子区域的图斑数尽量均匀

### 1.4 问题建模：从几何到图论

**核心思想**：将几何问题转化为图论问题。

构建**邻接图（Adjacency Graph）**：
- **节点** = 每个图斑
- **边** = 两个图斑共边（shared boundary）则连边
- **权重** = 可以用共边长度作为边权

一旦建立了邻接图，"切分村边界"就变成了"图划分（Graph Partitioning）"问题：
- 将图的节点分成若干组
- 每组节点数 ≤ 阈值K
- 各组尽量均衡
- 切边数尽量少（切边少 → 切割线短 → 子区域边界更规整）

## 第二部分：方案对比

### 方案A：坐标递归二分法（Coordinate Recursive Bisection）

**核心思想**：沿较长轴方向，按图斑质心坐标排序，从中位数处切分，递归处理。

**算法流程**：
```
function split(region, parcels, threshold):
    if |parcels| ≤ threshold: return
    axis = 较长轴方向(region的包围盒)
    按质心在axis上的坐标排序parcels
    mid = |parcels| / 2
    splitValue = parcels[mid]和parcels[mid-1]质心在axis上的中间值
    cutLine = 在splitValue处创建垂直于axis的切割线
    验证cutLine不穿过任何parcel，否则微调
    分割region为regionA, regionB
    递归 split(regionA, parcelsInA, threshold)
    递归 split(regionB, parcelsInB, threshold)
```

**优点**：实现简单，空间局部性好。
**缺点**：直线切割可能穿过图斑；微调策略可能复杂；不考虑图斑间的拓扑关系。
**复杂度**：O(N log²N)

### 方案B：基于邻接图的图划分（Graph Partitioning）

**核心思想**：建立图斑邻接图，使用图划分算法（如METIS的多层级方案）进行划分。

**算法流程**：
```
1. 建立空间索引（R-tree / STR-tree）
2. 对每个图斑，查询相邻图斑，建立邻接图
3. 使用图划分算法将节点分为⌈N/K⌉组
4. 对每组图斑，计算其在村边界内的几何区域
```

**优点**：理论最优的均衡性和切边最小化；天然不切割图斑。
**缺点**：需要引入图划分库或自行实现；从图划分结果恢复几何区域较复杂。
**复杂度**：O(N log N) 建图 + O(N) 多层级划分（METIS）

### 方案C：基于Voronoi的区域生长（Region Growing）

**核心思想**：选择K个种子点，每个种子点"生长"出一个区域，直到包含足够的图斑。

**优点**：区域形状自然。
**缺点**：种子选择敏感；均衡性难保证；可能产生不连通区域。
**复杂度**：O(N × K)

### 方案D：混合方案 —— 坐标递归二分 + 图斑感知切割（推荐）

**核心思想**：结合方案A的简单性和方案B对拓扑的尊重。

```
1. 建立空间索引
2. 递归二分：
   a. 选择切割轴（较长方向）
   b. 按质心排序，找到中位数切分点
   c. 生成初始切割线
   d. 检测切割线是否穿过图斑
   e. 如果穿过，将被穿过的图斑整体分配到一侧（基于质心位置）
   f. 用图斑边界"绕过"被影响的图斑，调整切割线
   g. 用调整后的切割线切分村边界多边形
3. 递归处理子区域
```

**这是本文推荐的主要方案，因为**：
- 实现复杂度适中
- 几何语义清晰（切割线有明确的物理含义）
- 利用JTS的成熟空间操作
- 天然保证子区域是合法多边形

## 第四部分：如何判断"最优"切分方案

### 4.1 量化指标

**均衡度（Balance Factor）**：
```
BF = max(|Pi|) / avg(|Pi|)
```
其中Pi是第i个子区域的图斑数。BF=1.0为完美均衡，越大越不均衡。

**切割代价（Cut Cost）**：
```
CC = Σ(切割线穿过的图斑共边数)
```
切割代价越低，说明切割线越少破坏图斑间的自然关系。

**紧凑度（Compactness）**：
```
Compactness_i = 4π × Area_i / Perimeter_i²
```
Polsby-Popper指标，1.0为完美圆形。越接近1越紧凑。

**面积均衡度**：
```
AF = max(Area_i) / avg(Area_i)
```

### 4.2 综合评分

```
Score = w1 × (1/BF) + w2 × (1/CC) + w3 × avg(Compactness) + w4 × (1/AF)
```

权重可根据实际业务需求调整。通常均衡度权重最高。

## 第五部分：核心概念与库的讲解

### 5.1 JTS Topology Suite —— 是什么，为什么用它

**是什么**：Java拓扑套件（Java Topology Suite），是Java平台最权威的计算几何库，实现了OGC（开放地理空间联盟）的Simple Features规范。

**为什么用它**：
- **工业标准**：PostGIS（PostgreSQL的空间扩展）的核心算法就来自JTS
- **拓扑操作完整**：交集、并集、差集、缓冲区、包含判断等
- **空间索引**：内置STR-tree（Sort-Tile-Recursive tree），对大量几何体的批量查询非常高效
- **稳健性**：使用了鲁棒的几何谓词（snap rounding等），能处理浮点精度问题

**核心类层次**：
```
Geometry (抽象基类)
  ├── Point          # 点
  ├── LineString     # 折线
  ├── LinearRing     # 闭合环
  ├── Polygon        # 多边形 = 外环 + 0或多个内环（洞）
  ├── MultiPoint     # 多点
  ├── MultiLineString # 多线
  ├── MultiPolygon   # 多面
  └── GeometryCollection # 几何集合
```

### 5.2 STR-tree（Sort-Tile-Recursive Tree）

**是什么**：一种批量加载的R-tree变体，专为"先全部插入，后批量查询"的场景优化。

**为什么用它而不是普通R-tree**：
- 普通R-tree在逐个插入时可能产生大量重叠的包围盒
- STR-tree先对所有几何体排序（Sort），然后分块（Tile），递归构建（Recursive）
- 结果是包围盒重叠极少，查询效率更高
- 我们的场景恰好是"所有图斑已知，需要大量邻接查询"，完美匹配

**复杂度**：构建O(N log N)，查询O(log N + k)，k为结果数。

### 5.3 图划分问题（Graph Partitioning）

**是什么**：将图的节点分成K组，使得：
- 各组大小尽量均衡
- 跨组的边数（cut edges）尽量少

**为什么切边少很重要**：
- 在我们的问题中，切边对应被切割线穿过的图斑共边
- 切边少 → 切割线短 → 子区域边界更规整
- 切边少 → 被分开的相邻图斑少 → 更符合地理直觉

**经典算法**：
- **Kernighan-Lin（KL）算法**：贪心交换节点对，O(N² log N)
- **谱方法（Spectral Bisection）**：用拉普拉斯矩阵的第二小特征向量（Fiedler向量），O(N^1.5)
- **多层级方法（Multilevel，如METIS）**：粗化→划分→细化，O(N)

### 5.4 递归二分法（Recursive Bisection）

**是什么**：将整体一分为二，然后对每半递归处理。

**为什么用递归二分而不是直接K路划分**：
- 每次只需解决一个二分问题，比直接K路划分简单得多
- 递归深度为O(log(N/K))，通常很浅
- 每层的总工作量为O(N)
- 总复杂度O(N log(N/K))

### 5.5 多边形切割（Polygon Split）

**是什么**：用一条线将一个多边形切成两个或多个部分。

**为什么不直接用JTS的intersection**：
- intersection(A, line)得到的是线段，不是多边形
- 需要用difference操作：regionA = region.difference(halfPlane)
- 或者用更精确的split操作：构造切割线两侧的半平面，分别与原多边形求交

**实现技巧**：

## 第六部分：复杂度分析

### 时间复杂度

| 步骤 | 复杂度 | 说明 |
|------|--------|------|
| 建空间索引 | O(N log N) | STR-tree构建 |
| 建邻接图 | O(N log N) | 每个图斑查询邻居 |
| 递归二分 | O(N log(N/K)) | log(N/K)层，每层O(N) |
| 几何切割 | O(N log(N/K)) | 每层切割操作 |
| **总计** | **O(N log N)** | 瓶颈在建索引和建图 |

### 空间复杂度

| 数据结构 | 复杂度 | 说明 |
|----------|--------|------|
| 图斑几何 | O(N × V) | V为平均顶点数 |
| 空间索引 | O(N) | STR-tree |
| 邻接图 | O(N + E) | E为边数 |
| 递归栈 | O(log(N/K)) | 递归深度 |
| **总计** | **O(N × V)** | 瓶颈在几何数据 |

对于N=100,000, K=1,000：递归深度约7层，非常可控。

## 第八部分：思维内化总结

### 核心思维模式

**1. 降维思考法**：
遇到复杂的二维/三维问题时，先考虑一维情形怎么解，找到核心约束和核心操作，再推广到高维。本题中，一维的"在间隙处切割"是所有方案的基础直觉。

**2. 域转换法**：
几何域的问题如果直接处理会非常复杂（浮点精度、拓扑异常等），转换到图论域后，很多约束变成了图划分的标准约束。关键是要能准确地在两个域之间转换（图斑→节点，共边→边，切割线→切边）。

**3. 分治递归法**：
面对大规模问题（10万级），不要试图一步到位。递归二分每次只处理一个简单的"一分为二"决策，log层就能解决问题。这是分治策略的核心。

**4. 约束松弛法**：
先忽略"切割线不能穿过图斑"这个硬约束，用简单的坐标二分得到初始方案。然后"修复"违反约束的部分（将被穿过的图斑整体分配到一侧）。先求解松弛问题再修复，比直接求解完整问题简单得多。

### 迁移应用

这套思维方法可以应用到很多类似问题：
- **大规模地图瓦片生成**：将地图数据按区域切分，每个瓦片的要素数不超过阈值
- **分布式空间计算**：将空间数据分片到不同计算节点
- **行政区划优化**：重新划分选区使人口均衡
- **物流配送区域划分**：将配送区域切分使每个区域的订单量均衡
