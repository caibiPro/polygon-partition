# 第 1 课：从零写出“连通分量”

## 一、这节课学完，你要真正会什么

你要能独立完成这三件事：

1. 用邻接表表示一个无向图
2. 用 DFS 或 BFS 找出所有连通分量
3. 判断一组节点是不是连通的

这三件事对你的业务分别对应：

- 图斑 = 节点
- 共边关系 = 边
- 一组图斑是不是一整块 = 这一组节点是不是连通

所以这不是“单纯学图论”，而是在写你业务问题的最小内核。

---

## 二、先记住今天只需要的 4 个概念

先不要背大段定义，只记这 4 个。

**节点**
图里的一个点。
在你的业务里，对应一个图斑。

**边**
两个节点之间的连接。
在你的业务里，对应两个图斑共边。

**邻接表**
“每个节点都记录自己连着谁”的存储方式。

**连通分量**
图里一团彼此能走到的节点集合。
如果两团节点互相走不到，那就是两个连通分量。

你今天所有代码都围绕这四个词。

先做一个纯图程序，输入这种关系：

## 作业 1：先定义图结构

你先写一个 `Graph` 类，要求：

- 节点编号从 `0` 到 `n-1`
- 用邻接表存图
- 这是一个**无向图**
- 要能加边
- 要能拿到某个节点的邻居列表

你先自己设计，但我建议先用这个骨架：

```java
import java.util.*;

public class Graph {
    private final List<List<Integer>> adj;

    public Graph(int n) {
        // 你来实现
    }

    public void addUndirectedEdge(int u, int v) {
        // 你来实现
    }

    public List<Integer> neighbors(int u) {
        // 你来实现
        return null;
    }

    public int size() {
        // 你来实现
        return 0;
    }
}
```

### 你这一步必须达到的要求

1. `new Graph(6)` 能创建 6 个节点
2. `addUndirectedEdge(0, 1)` 后，0 的邻居里有 1，1 的邻居里也有 0
3. 不要允许非法下标 silently 通过
4. 先不考虑去重，允许你后面再优化

### 这一步你要想清楚的问题

为什么这里用 `List<List<Integer>>`，而不是矩阵？

因为你后面的图斑邻接图通常是**稀疏图**。
每个图斑不会和几千个图斑相邻，通常只和少数几个相邻。
所以邻接表比邻接矩阵更合适。

## 作业 2：写 DFS，找到一个起点能到达的所有节点

你先不要急着写“所有连通分量”。
先写一个小方法：

给定起点 `start`，把和它连通的所有节点找出来。

我建议你先写**迭代版 DFS**，不要先写递归版。
原因很简单：你以后会面对 1 万、10 万节点，递归深了容易栈溢出。迭代版更稳。

你可以先设计成这样：

```java
public static List<Integer> dfsComponent(Graph graph, int start, boolean[] visited) {
    // 你来实现
    return null;
}
```

### 要求

1. 用 `visited` 避免重复访问
2. 返回这个起点所在的整块节点
3. 返回结果建议按访问顺序即可，不要求排序
4. 允许你用 `Stack` 或 `Deque`

### 提示

你很可能会写到这几个结构：

```java
Deque<Integer> stack = new ArrayDeque<>();
stack.push(start);
```

然后循环：

- 弹出一个节点
- 如果已经访问过就跳过
- 否则标记访问
- 把它的邻居压栈

### 这一小步在业务里对应什么

它对应的是：

“从一个图斑开始，把和它共边连成一整片的图斑全找出来。”

这已经非常接近你的真实问题了。

## 作业 3：写出所有连通分量

有了前两步，这一步就很自然。

方法签名你可以自己定，我建议这样：

```java
public static List<List<Integer>> connectedComponents(Graph graph) {
    // 你来实现
    return null;
}
```

### 思路

- 准备一个 `visited[]`
- 从 `0` 遍历到 `n-1`
- 每碰到一个没访问过的节点
  - 调一次 `dfsComponent(...)`
  - 得到一个新的连通分量
  - 加入结果列表

### 要求

1. 每个节点必须恰好出现在一个分量里
2. 孤立点也要作为单独分量返回
3. 最终返回 `List<List<Integer>>`

### 建议

为了方便测试，你可以在返回前对每个分量内部排序，再按分量第一个元素排序。
这样输出会更稳定，便于你自己核对。

这不是必须，但对调试很有帮助。

## 作业 4：写一个“子集是否连通”的方法

这个方法和你后面的业务联系最紧。

你可以设计成：

```java
public static boolean isConnectedSubset(Graph graph, Set<Integer> subset) {
    // 你来实现
    return false;
}
```

### 含义

不是判断整张图是否连通，
而是判断“某一组指定节点”在这张图里是不是连成一片。

### 为什么这一步重要

因为以后你的一个切分结果，本质上就是“某一组图斑”。
判断这个切分结果是不是一个完整区域，底层就是：

**这组图斑在邻接图里是不是连通。**

### 提示

你只需要：

1. 从 `subset` 里随便找一个起点
2. 只在 `subset` 内部做 DFS/BFS
3. 最后看能访问到的节点数是否等于 `subset.size()`

### 边界情况

`subset` 为空时，你要自己定义。
我建议你先返回 `true`，因为“空集连通性”在工程里通常不会成为问题，而且这样代码更顺。

## 九、你做的时候最容易犯的 6 个错误

你可以提前防一下。

第一，`addUndirectedEdge` 只加了一边。
结果图变成了有向图，后面连通性全乱。

第二，访问时没有及时标记 `visited`。
会导致节点反复入栈甚至死循环。

第三，遍历邻居时忘了判断 `visited`。
结果重复访问很多次。

第四，把“整图是否连通”和“子集是否连通”混了。
这两个不是一回事。

第五，`subset` 检查时把不在 `subset` 的邻居也走进去了。
这样会把外部路径算进来，结果错误。

第六，没有处理孤立点。
孤立点也是合法连通分量。
